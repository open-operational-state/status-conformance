# Compatibility

This directory will contain compatibility matrices documenting how the standard interoperates with existing systems, formats, and platforms.

## Purpose

Compatibility matrices track:

- Which existing formats can be mapped to the standard (and at what fidelity)
- Which platforms and frameworks are supported by adapters
- Which features are available across different serializations and profiles
- Known limitations or incompatibilities

## Status

Compatibility matrices will be created as the specification and adapter ecosystem matures.
